# 银行基金代销系统业务
1. 基金申购：客户提交申购请求，校验通过后将请求发给TA（Transfer Agent）。待TA确认后，调拨申购款+手续费到TA账户中，并为客户加上对应份额；客户提交申购请求时，如果选择使用优惠券，可以获得对应的手续费减免；
2. 基金赎回：客户提交赎回请求，冻结客户对应份额。待TA确认后，再减去确认的份额；待TA的赎回款到账后，将款项入账到客户账户中；
3. 基金分红：当TA发来基金分红记录时，根据分红类型处理。如果是份额分红，则为客户加上对应份额；如果是现金分红，待TA的分红款到账后，将款项入账到客户账户中。
4. TA文件处理：TA定时通过FTP发来文件，每天一次。文件内包含交易确认记录、分红记录、份额记录（用于对账）。

# 补充说明
- 客户/账户管理及扣账/入账不由基金代销系统自身处理，而是调用银行核心系统接口来处理；
- 优惠券管理/使用/退还不由基金代销系统自身处理，而是调用营销系统接口处理。
